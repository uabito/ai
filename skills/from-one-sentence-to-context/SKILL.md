# From One Sentence To Context

## Core Principle

Do not rush to the final deliverable. First help the user clarify the task, audience, goal, constraints, risks, context, and quality bar.

The goal is to move from a vague initial thought to a clear brief that can later produce a strong result.

## Default Workflow

1. Restate the user's initial thought in clearer business language.
2. Identify what is missing before a high-quality result can be produced.
3. Ask targeted questions to extract missing context.
4. Wait for answers before creating the final output.
5. Summarize the clarified brief.
6. Propose 2-3 possible structure or storyline directions.
7. Explain when each direction is best suited.
8. Ask the user to choose or adjust a direction.
9. Only after the direction is chosen, help create the first structured output.

## Question Areas to Cover

Ask questions that clarify:

- Audience: who will read, hear, or use the result?
- Goal: what should the audience understand, decide, believe, or do?
- Context: what background, data, history, constraints, and assumptions matter?
- Output type: presentation, memo, proposal, analysis, communication, plan, or another deliverable.
- Quality level: quick draft, working version, or finished result.
- Usage: live presentation, document for reading, internal discussion, external client material, board-level decision, or follow-up artifact.
- Constraints: time, format, tone, length, sensitivity, confidentiality, brand expectations, and required sources.
- Risks: what could be misunderstood, challenged, oversimplified, or politically sensitive?
- Human decisions: which choices must remain with the user instead of being guessed by AI?

## Interaction Rules

- Do not create generic content from a vague prompt.
- Do not produce the final result before asking clarifying questions, unless the user explicitly asks for a quick draft.
- Use the user's language for the response unless they ask for another language.
- Ask questions in a practical, business-friendly tone.
- Keep questions grouped and manageable. Prefer 7-10 strong questions over a long questionnaire.
- If the user does not know an answer, mark it as an open question and suggest how to proceed.
- If the task is too broad, propose narrower versions.
- If the desired output type is unclear, ask how the result will be used.
- If the user wants a polished result, spend more effort on context, structure, and iteration.
- If the user wants speed, state what assumptions you are making.

## Output Format

When applying this skill, use this structure by default:

1. **Clarified Starting Point**
2. **What Is Missing**
3. **Context Questions**
4. **Clarified Brief Summary** after the user answers
5. **2-3 Structure Directions**
6. **Recommended Next Step**
